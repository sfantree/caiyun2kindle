Installation guide

安装指南

From ArchWiki
Jump to navigation 跳转到导航栏 Jump to search 跳转到搜索

This document is a guide for installing Arch Linux from the live system booted with the official installation image. Before installing, it would be advised to view the FAQ. For conventions used in this document, see Help:Reading. In particular, code examples may contain placeholders (formatted in italics) that must be replaced manually.

本文档是使用官方安装映像从实时系统启动安装 Arch Linux 的指南。 在安装之前，建议查看 FAQ。 有关本文档中使用的约定，请参阅帮助: 阅读。 特别是，代码示例可能包含占位符(格式为斜体) ，必须手动替换。

For more detailed instructions, see the respective ArchWiki articles or the various programs' man pages, both linked from this guide. For interactive help, the IRC channel and the forums are also available.

有关更详细的说明，请参阅各自的 ArchWiki 文章或各种程序的手册页，这两者都链接自本指南。 对于交互式帮助，IRC 频道和论坛也是可用的。

Arch Linux should run on any x86_64-compatible machine with a minimum of 512 MiB RAM. A basic installation should take less than 800 MiB of disk space. As the installation process needs to retrieve packages from a remote repository, this guide assumes a working internet connection is available.

Arch Linux 应该在至少512 MiB RAM 的任何 x8664兼容机器上运行。 一个基本的安装应该占用不到800mib 的磁盘空间。 由于安装过程需要从远程存储库检索包，因此本指南假定有可用的 internet 连接。

Pre-installation 安装前

The installation media and their GnuPG signatures can be acquired from the Download page.

可以从下载页获取安装媒体及其 GnuPG 签名。

Verify signature 验证签名

It is recommended to verify the image signature before use, especially when downloading from an HTTP mirror, where downloads are generally prone to be intercepted to serve malicious images.

建议在使用之前验证镜像签名，尤其是从 HTTP 镜像下载时，因为下载通常很容易被拦截以提供恶意镜像。

On a system with GnuPG installed, do this by downloading the PGP signature (under Checksums) to the ISO directory, and verifying it with:

在安装了 GnuPG 的系统上，可以通过下载 PGP 签名(在 Checksums 下)到 ISO 目录，并用以下方法进行验证:

$ gpg --keyserver-options auto-key-retrieve --verify archlinux-version-x86_64.iso.sig

Alternatively, from an existing Arch Linux installation run:

或者，从现有的 Arch Linux 安装运行:

$ pacman-key -v archlinux-version-x86_64.iso.sig
Note: 注意:
  • The signature itself could be manipulated if it is downloaded from a mirror site, instead of from 如果签名是从镜像站点下载的，而不是从镜像站点下载的，那么签名本身就可以被操纵archlinux.org as above. In this case, ensure that the public key, which is used to decode the signature, is signed by another, trustworthy key. The 如上所述。 在这种情况下，确保用于解码签名的公钥由另一个值得信赖的密钥签名。 这个gpg command will output the fingerprint of the public key. 命令会输出公钥的指纹
  • Another method to verify the authenticity of the signature is to ensure that the public key's fingerprint is identical to the key fingerprint of the 另一个验证签名真伪的方法是确保公用钥匙的指纹与Arch Linux developer 开发者 who signed the ISO-file. See 是谁签署了 iso 文件看Wikipedia:Public-key cryptography 维基百科: 公开密钥加密 for more information on the public-key process to authenticate keys. 获取有关公开密钥进程的更多信息，以验证密钥

Boot the live environment 启动生活环境

The live environment can be booted from a USB flash drive, an optical disc or a network with PXE. For alternative means of installation, see Category:Installation process.

Live 环境可以从 USB 闪存驱动器、光盘或 PXE 网络启动。 有关其他安装方式，请参阅类别: 安装过程。

  • Pointing the current boot device to a drive containing the Arch installation media is typically achieved by pressing a key during the 将当前引导设备指向包含 Arch 安装介质的驱动器通常是通过在POST 邮报 phase, as indicated on the splash screen. Refer to your motherboard's manual for details. 阶段，如启动画面所示。详细信息请参阅主板手册
  • When the Arch menu appears, select 当 Arch 菜单出现时，选择Boot Arch Linux 启动拱 Linux and press 然后按Enter to enter the installation environment. 进入安装环境
  • See 看README.bootparams 说明: bootparams for a list of 为了一份boot parameters 启动参数, and ，及packages.x86_64 Packages.x8664 for a list of included packages. 包裹清单
  • You will be logged in on the first 你将在第一个登录virtual console 虚拟控制台 as the root user, and presented with a 作为根用户，并显示一个Zsh shell prompt. 外壳提示符

To switch to a different console—for example, to view this guide with ELinks alongside the installation—use the Alt+arrow shortcut. To edit configuration files, nano, vi and vim are available.

要切换到不同的控制台ーー例如，在安装时使用 ELinks 查看本指南ーー请使用 Alt + 箭头快捷键。 要编辑配置文件，可以使用 nano、 vi 和 vim。

Set the keyboard layout 设置键盘布局

The default console keymap is US. Available layouts can be listed with:

默认的控制台键盘图是美元。可用的布局可以通过以下方式列出:

# ls /usr/share/kbd/keymaps/**/*.map.gz

To modify the layout, append a corresponding file name to loadkeys(1), omitting path and file extension. For example, to set a German keyboard layout:

要修改布局，在 loadkeys (1)中附加相应的文件名，省略路径和文件扩展名。 例如，设置一个德语键盘布局:

# loadkeys de-latin1

Console fonts are located in /usr/share/kbd/consolefonts/ and can likewise be set with setfont(8).

控制台字体位于 / usr / share / kbd / consolefonts / 中，也可以使用 setfont (8)进行设置。

Verify the boot mode 验证启动模式

If UEFI mode is enabled on an UEFI motherboard, Archiso will boot Arch Linux accordingly via systemd-boot. To verify this, list the efivars directory:

如果在 UEFI 主板上启用了 UEFI 模式，Archiso 将相应地通过 systemd-boot 启动 Arch Linux。 为了验证这一点，列出 efivars 目录:

# ls /sys/firmware/efi/efivars

If the directory does not exist, the system may be booted in BIOS or CSM mode. Refer to your motherboard's manual for details.

如果目录不存在，系统可能以 BIOS 或 CSM 模式启动。 详细信息请参阅主板手册。

Connect to the internet 连接到互联网

To set up a network connection, go through the following steps:

要建立网络连接，需要执行以下步骤:

  • Ensure your 确保你的network interface 网络接口 is listed and enabled, for example with 列出并启用，例如使用ip-link(8) Ip-link (8): 
    # ip link
  • Connect to the network. Plug in the Ethernet cable or 连接到网络。插入以太网电缆或connect to the wireless LAN 连接到无线局域网.
  • Configure your network connection: 配置您的网络连接:
    • Static IP address 静态 IP 地址
    • Dynamic IP address: use 动态 IP 地址: 使用DHCP 主机配置协议.
    Note: 注意: The installation image enables 安装映像启用dhcpcd (dhcpcd@interface.service) for )免费下载该刊物wired network devices 有线网络设备 on boot. 穿靴
  • The connection may be verified with 这个连接可以用ping 平: 
    # ping archlinux.org

Update the system clock 更新系统时钟

Use timedatectl(1) to ensure the system clock is accurate:

使用 timedatectl (1)以确保系统时钟准确:

# timedatectl set-ntp true

To check the service status, use timedatectl status.

若要检查服务状态，请使用 timedatectl 状态。

Partition the disks 对磁盘进行分区

When recognized by the live system, disks are assigned to a block device such as /dev/sda or /dev/nvme0n1. To identify these devices, use lsblk or fdisk.

当活动系统识别磁盘时，磁盘被分配给一个块设备，比如 / dev / sda 或 / dev / nvme0n1。 要识别这些设备，请使用 lsblk 或 fdisk。

# fdisk -l

Results ending in rom, loop or airoot may be ignored.

以 rom、 loop 或 airoot 结尾的结果可能被忽略。

The following partitions are required for a chosen device:

对于所选择的设备，需要以下分区:

  • One partition for the root directory 根目录的一个分区/.
  • If 如果UEFI is enabled, an 启用，一个EFI system partition EFI系统分区.

If you want to create any stacked block devices for LVM, system encryption or RAID, do it now.

如果您想为 LVM、系统加密或 RAID 创建任何堆叠块设备，请现在就创建。

Format the partitions 格式化分区

Once the partitions have been created, each must be formatted with an appropriate file system. For example, if the root partition is on /dev/sdX1 and will contain the ext4 file system, run:

创建分区之后，必须使用适当的文件系统对每个分区进行格式化。 例如，如果根分区在 / dev / sdx1上，并且包含 ext4文件系统，那么运行:

# mkfs.ext4 /dev/sdX1

If you created a partition for swap, initialize it with mkswap:

如果你为 swap 创建了一个分区，使用 mkswap 初始化它:

# mkswap /dev/sdX2
# swapon /dev/sdX2

See File systems#Create a file system for details.

有关详细信息，请参阅文件系统 # create a File system。

Mount the file systems 装入文件系统

Mount the file system on the root partition to /mnt, for example:

将根分区上的文件系统挂载到 / mnt，例如:

# mount /dev/sdX1 /mnt

Create any remaining mount points (such as /mnt/efi) and mount their corresponding partitions.

创建任何剩余的挂载点(例如 / mnt / efi)并挂载它们相应的分区。

genfstab will later detect mounted file systems and swap space.

稍后，genfstab 将检测挂载的文件系统和交换空间。

Installation 安装

Select the mirrors 选择镜子

Packages to be installed must be downloaded from mirror servers, which are defined in /etc/pacman.d/mirrorlist. On the live system, all mirrors are enabled, and sorted by their synchronization status and speed at the time the installation image was created.

要安装的包必须从镜像服务器下载，这些服务器在 / etc / pacman 中定义。 镜子清单。 在实时系统上，启用所有镜像，并根据创建安装镜像时的同步状态和速度进行排序。

The higher a mirror is placed in the list, the more priority it is given when downloading a package. You may want to edit the file accordingly, and move the geographically closest mirrors to the top of the list, although other criteria should be taken into account.

镜像在列表中的位置越高，下载包时给它的优先级就越高。 您可能希望相应地编辑文件，并将地理上最接近的镜像移到列表的顶部，尽管应该考虑其他条件。

This file will later be copied to the new system by pacstrap, so it is worth getting right.

这个文件稍后会被 pacstrap 复制到新的系统中，所以它值得正确使用。

Install essential packages 安装必要的软件包

Use the pacstrap script to install the base package, Linux kernel and firmware for common hardware:

使用 pacstrap 脚本安装常用硬件的基本包、 Linux 内核和固件:

# pacstrap /mnt base linux linux-firmware
Tip: 提示: You can substitute 你可以替换linux for a 为了kernel 核 package of your choice. You can omit the installation of the kernel or the firmware package if you know what you are doing. 你自己选吧。 如果知道要做什么，可以省略内核或固件包的安装

The base package does not include all tools from the live installation, so installing other packages may be necessary for a fully functional base system. In particular, consider installing:

基础软件包不包括来自现场安装的所有工具，因此对于功能齐全的基础系统来说，可能需要安装其他软件包。 特别是，考虑安装:

  • userspace utilities for the management of 用户空间管理工具file systems 文件系统 that will be used on the system, 将会在系统中使用,
  • utilities for accessing 用于访问的实用程序RAID 磁盘阵列 or 或LVM 单板层积材 partitions, 分区,
  • specific firmware for other devices not included in 其他未包括在内的设备的特定固件linux-firmware Linux 固件,
  • software necessary for 必要的软件networking 网络,
  • a text editor 文本编辑程序,
  • packages for accessing documentation in 用于访问文档的包man 男人 and 及info 信息 pages: 页:man-db 人声, man-pages 手册页 and 及texinfo.

To install other packages or package groups, append the names to the pacstrap command above (space separated) or use pacman while chrooted into the new system. For comparison, packages available in the live system can be found in packages.x86_64.

若要安装其他软件包或软件包组，请将名称附加到上面的 pacstrap 命令(空格分隔) ，或者在新系统中使用 pacman。 为了便于比较，可以在 packages.x8664中找到活动系统中可用的包。

Configure the system 配置系统

Fstab

Generate an fstab file (use -U or -L to define by UUID or labels, respectively):

生成一个 fstab 文件(分别使用-u 或-l 通过 UUID 或标签定义) :

# genfstab -U /mnt >> /mnt/etc/fstab

Check the resulting /mnt/etc/fstab file, and edit it in case of errors.

检查结果 / mnt / etc / fstab 文件，并在出现错误时对其进行编辑。

Chroot

Change root into the new system:

将 root 更改为新系统:

# arch-chroot /mnt

Time zone 时区

Set the time zone:

设定时区:

# ln -sf /usr/share/zoneinfo/Region/City /etc/localtime

Run hwclock(8) to generate /etc/adjtime:

运行 hwclock (8)生成 / etc / adjtime:

# hwclock --systohc

This command assumes the hardware clock is set to UTC. See System time#Time standard for details.

此命令假定硬件时钟设置为 UTC。详情请参阅系统时间 # time 标准。

Localization 本地化

Edit /etc/locale.gen and uncomment en_US.UTF-8 UTF-8 and other needed locales. Generate the locales by running:

编辑 / etc / locale. gen 并在 us. UTF-8 UTF-8和其他需要的区域设置中取消注释。通过运行生成区域设置:

# locale-gen

Create the locale.conf(5) file, and set the LANG variable accordingly:

创建 locale.conf (5)文件，并相应地设置 LANG 变量:

/etc/locale.conf
LANG=en_US.UTF-8

If you set the keyboard layout, make the changes persistent in vconsole.conf(5):

如果您设置了键盘布局，那么可以在 vconsole.conf (5)中保持更改:

/etc/vconsole.conf
KEYMAP=de-latin1

Network configuration 网络配置

Create the hostname file:

创建主机名文件:

/etc/hostname
myhostname

Add matching entries to hosts(5):

向主机添加匹配条目(5) :

/etc/hosts
127.0.0.1	localhost
::1		localhost
127.0.1.1	myhostname.localdomain	myhostname

If the system has a permanent IP address, it should be used instead of 127.0.1.1.

如果系统有一个永久的 IP 地址，应该使用它而不是127.0.1.1。

Complete the network configuration for the newly installed environment, that includes installing your preferred network management software.

完成新安装环境的网络配置，包括安装首选的网络管理软件。

Initramfs

Creating a new initramfs is usually not required, because mkinitcpio was run on installation of the kernel package with pacstrap.

通常不需要创建新的 initramfs，因为 mkinitcpio 是在使用 pacstrap 安装内核包时运行的。

For LVM, system encryption or RAID, modify mkinitcpio.conf(5) and recreate the initramfs image:

对于 LVM、系统加密或 RAID，修改 mkinitcpio.conf (5)并重新创建 initramfs 映像:

# mkinitcpio -P

Root password 根密码

Set the root password:

设置 root 密码:

# passwd

Boot loader 引导加载程序

Choose and install a Linux-capable boot loader. If you have an Intel or AMD CPU, enable microcode updates in addition.

选择并安装一个支持 linux 的引导加载程序。如果您使用的是 Intel 或 AMD CPU，请另外启用微代码更新。

Reboot 重启

Exit the chroot environment by typing exit or pressing Ctrl+d.

通过键入 Exit 或按 Ctrl + d 退出 chroot 环境。

Optionally manually unmount all the partitions with umount -R /mnt: this allows noticing any "busy" partitions, and finding the cause with fuser(1).

可以选择使用 umount-r / mnt 手动卸载所有分区: 这允许注意到任何“忙碌”分区，并查找 fuser (1)的原因。

Finally, restart the machine by typing reboot: any partitions still mounted will be automatically unmounted by systemd. Remember to remove the installation media and then login into the new system with the root account.

最后，通过键入 reboot 重新启动计算机: 任何仍然挂载的分区将由 systemd 自动卸载。 请记住删除安装媒体，然后使用 root 帐户登录到新系统。

Post-installation 安装后

See General recommendations for system management directions and post-installation tutorials (like setting up a graphical user interface, sound or a touchpad).

请参阅系统管理方向和安装后教程的一般建议(如设置图形用户界面、声音或触摸板)。

For a list of applications that may be of interest, see List of applications.

有关可能令人感兴趣的申请的清单，请参阅申请清单。
